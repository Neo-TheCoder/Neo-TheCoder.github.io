进程间通信

Posix 和 System V
Unix OS操作风格上主要分为System V和BSD(Berkeley Software Distribution)
Linux的操作风格介于二者之间

Posix是Portable Operating System Interface是一系列标准，目的是为运行在不同操作系统的应用程序提供统一的接口，实现者是不同的操作系统内核

注意linux的进程间通信中，有posix信号量和System V信号量，前者是基于共享内存的，后者是基于内核的（需要进入内核态），多线程编程中一般不用基于内核的System信号量，因为线程相对于进程是轻量级的，从OS的调度开销角度看，每次使用该信号量就要进入内核态的话，丧失了线程的轻量优势。

进程间通信

1、进程间通信的目的

数据传输 资源共享 通知事件 进程控制

2、进程间通信的本质

让不同的进程看到同一份资源（内存、文件、内核缓冲）

资源由谁提供，就有了不同的进程间通信方式
文件-管道、OS内核IPC提供-SystemV IPC、网络-套接字

进程间的通信，要借助OS的资源

3、进程间通信分类

消息队列

System V IPC

System V 消息队列
System V 共享内存
System V 信号量

POSIX IPC

消息队列
共享内存
信号量
互斥量
条件变量
读写锁

管道

UNIX中最古老的进程间通信的形式

匿名管道

常用于父子进程间的通信
管道虽然用的是文件，但OS一定不会把进程进行通信的数据刷新到磁盘中，因为IO参与降低了效率

pipe函数和fork()函数搭配

因为是半双工，所以必须关掉一端，打开一端
从管道写端写入的数据会存在于内核缓冲，直到数据从管道中被读取
管道只是单向的通信，进程运行具有独立性，写时拷贝
PS：写时拷贝：等到修改数据时才真正分配内存空间 目的是延迟甚至是避免（不必要）的内存拷贝

举例：
在linux系统中，调用fork系统调用创建子进程时，并不会把父进程所有占用的内存页复制一份，而是与父进程共用相同的内存页，而当子进程或者父进程堆内存页修改时才会复制

原理是引用计数

匿名管道：
int pipe2(int pipefd[2],int flags);

管道内部自带同步互斥机制

一次只允许一个进程使用的资源，被称为临界资源
管道就是一种临界资源

内核会对管道操作进行同步和互斥

同步是更为复杂的互斥、互斥是一种特殊的同步

流式服务：
数据没有明确的分割，一次拿多少数据都行

与之相对的是数据报服务，数据有明确的分割，拿数据按报文段拿。

如何理解阻塞挂起，唤醒？
进程先立即停止执行，然很将PCB的状态改为阻塞状态，并将PCB插入相应的阻塞队列。
当结束阻塞的事件发生，阻塞进程从阻塞队列中移出，将其PCB的状态改为就绪状态，然后将PCB插入到就绪队列中去。

没有读者的写者会被OS直接杀掉。

有名管道

用于实现两个毫不相关的进程之间的通信
就是一种特殊类型的文件，两个进程通过命名管道打开同一个管道文件
命名管道和匿名管道一样都是内存文件，只是命名管道在磁盘有一个简单的影像，大小永远为0，这些数据不会被刷新到磁盘中

int mkfifo(const char *pathname, mode_t mode);
文件路径/文件名，管道文件的默认权限

PID: 进程标识符

PPID: 当前进程的父进程ID

管道文件的大小始终为0 通信是存在于内存中的

命名管道和匿名管道的区别
匿名管道由pipe函数创建并打开。
命名管道由mkfifo函数创建，由open函数打开。
FIFO（命名管道）与pipe（匿名管道）之间唯一的区别在于它们创建与打开的方式不同，一旦这些工作完成之后，它们具有相同的语义。

system V IPC提供的通信方式有以下三种：

system V共享内存
system V消息队列
system V信号量

system V共享内存和system V消息队列是以传送数据为目的的，而system V信号量是为了保证进程间的同步与互斥而设计的

System V 共享内存

共享内存：申请一块物理内存空间，将这块内存空间分别与需要进行进程间通信的进程的页表之间建立映射，再在虚拟地址空间中开辟空间，将虚拟地址填充到各自页表的对应位置，使得虚拟地址和物理地址之间建立起对应关系。

*** 本质：修改页表，虚拟地址中开辟空间 ***
开辟物理空间、虚拟地址、建立映射都是调用OS接口完成的

*** 共享内存的建立 ***
1、在物理内存中申请共享内存空间
2、将申请到的共享内存挂接到地址空间，建立映射

*** 共享内存的释放 ***
1、取消映射关系
2、释放共享内存空间

int shmget(key_t key, size_t size, int shmflg);

底层const：
当执行对象的拷贝操作时，拷入和拷出的对象必须具有相同的底层const资格，或者两个对象的数据类型可以转换。

底层const的限制不能忽视：

或者数据类型可以转换

p2 = &i; // int 可以转为const int

auto类型说明符

decltype类型指示符
希望从表达式的类型推断出腰定义的变量的类型，但是不想用该表达式的值初始化变量
作用是：选择并返回操作数的数据类型

decltype保留顶层const
int i=42,*p=&i,&r=i;

解引用指针可以得到指针所指的对象，因此得到的是引用

int i= 42;

双层括号，编译器把它当成表达式，是可以作为赋值语句左值的特殊表达式 总之双层括号的结果永远是引用

数据抽象依赖于接口和实现分离。
类的接口包括用户所能执行的操作;类的实现包括类的数据成员、负责接口实现的函数体及定义类所需的各种私有函数。

封装实现了类的接口和实现的分离，封装后的类隐藏了实现细节。
类的用户只能使用接口而无法访问实现部分。

*this代表的是执行该函数的对象

7.1.4 构造函数

只要类的对象被创建，就会执行构造函数。
不能是const的，构造函数在const对象的构造过程中可以向其写入值，因为直到构造函数完成初始化过程，对象才能真正取得常量属性。

默认构造函数是有缺陷的：没有赋给成员初值

*** 初始值列表 ***
Sales_data(const std::string &s,unsigned n,double p):book(s),units_sold(n),revenue(p*n) {}

局部对象会在创建它的块结束时被销毁
使用vector或者string的类能避免分配和释放内存带来的复杂性

把数据成员的访问权限设成private，外层就无法访问，当修改了类的接口，用户代码就无需改变，而且也可以防止用户的原因造成数据被破坏。

定义在类内部的成员函数自动inline
本质上减少了汇编:函数调用的开销(汇编call指令、清理现场:将压入栈的数据都出栈)
像宏一样展开，相比宏替换，编译器对待内联函数会检查参数类型，而宏替换不会，有安全隐患

mutable关键字，用于能修改类的某个数据成员，声明一个可变数据成员（即便是const对象的成员）。

类数据成员的初始值
提供类内初始值必须以’=’或者花括号表示

可以根据对象是否是const重载函数

7.3.3 类类型

前向声明：先声明类而不定义

7.4 类的作用域

在类的作用域外，普通的数据和函数成员只能由对象、引用或者指针使用成员访问运算符来访问。

注意：如果类内部的成员使用了外层作用域中的某个名字，而该名字代表一种类型，则类不能在之后重新定义该名字。

类型名的定义通常在类的开始处，确保所有使用该类型的成员都出现在类名的定义之后。

7.5 构造函数再探

const或者引用必须初始化

初始化和赋值差别在于底层的效率问题，前者直接初始化，后者先初始化再赋值

X(int val):j(val),i(j){} // 实际上先初始化i，再初始化j

写成这样更好：i (val),j(val)

7.5.2 委托构造函数

一个委托构造函数使用它所属类的其他构造函数执行它自己的初始化过程

A obj(); // 声明的是函数而非对象

7.5.4 隐式的类类型转换

只允许一步类型转换
string n = “999”;
item.combine(n);

或者item.combine(string(“999”));

在要求隐式转换的程序上下文中，可以通过将构造函数声明为explicit阻止隐式构造

explicit构造函数只能用于直接初始化

7.5.5 聚合类

表示可以直接访问成员、并且具有特殊的初始化语法形式

constexpr构造函数用于生成constexpr对象

7.6 类的静态成员

static 关键字使得类的成员与类关联在一起（而非对象）
类的静态成员实际上存储在任何对象之外，不包含this指针
不是由类的构造函数初始化

eg double r;
r = Account::rate();

关于人生的意义

发表于 2023-01-01 分类于 生活
本文字数： 666 阅读时长 ≈ 1 分钟

人为什么而活？人活着是为了什么？人生的意义在哪里？这个问题对于有智慧的人来说，是一个经典且难以回答的问题。
每个人对它的回答会带有其世界观、人生观的投射。

我经常思考这个问题。我确实经常觉得人活着没什么意思，“没意思”就是没有搞明白为什么而活，并且对这个问题感到极度的困惑。或许是肉体上的残缺以及精神上遭遇的挫折，经常觉得快乐的时候少，而痛苦悲伤的时候多，感觉到“热闹是他们的，我什么也没有”的痛苦。
但是想到家中父母亲人就会为自己有这种念头而惶恐。
但是我终究更多会产生积极的想法（没有积极的想法我肯定就不在了）：人活着必然是有意义的，因为我看过急诊室故事，知道了生命的可贵。诚然，生命绝对是可贵的，我小时候曾思考人死后会如何，想到失去生命后的无尽空虚竟惶恐地流下眼泪。人活着必须要珍惜生命，不珍惜生命是可耻的。“去跑！去跳！去做一个漂亮的倒挂金钩吧！”刚看完这个纪录片我的思想会很豁达，觉得生活中那些鸡毛蒜皮根本不算事，但终究还是会回到鸡毛蒜皮上来。
总之，我还是无法想到一个很好、很完美的回答。让思路活跃点————
在人脑具有高度发达的智慧以前，也就是原始人时期，人是不会思考人生的意义，人的行为和活动由本能驱使。
当人脑更有智慧，演化出社会的概念后，人必然会做的事情就是思考为什么而活。

我记得一个戏谑的回答是“生活就是生下来，然后活下去”。

古时说“立功、立德、立言”，背后是潜在的道德规范在赋予意义。

意义是本无所谓有，无所谓无的，因为意义都是人赋予的。

我们不能指望任何一个智者给我们答案，我们只有在生活的过程中以自己的方式尝试着找到答案。

我的2022

前言

Don’t look back in anger
现在是2022年12月31日晚上的23点48分，2023即将到来。
2022对我来说是极为特殊的一年，这一年我思考了未来，思考了人生。无论如何，总算是成长了一些，从人的角度来看。觉得有必要将我的思考和疑问记录下来，托付给未来的我继续思考。
姑且就按照手机相册里那些照片的顺序来回忆吧。希望途中慢慢酝酿出感悟，顺便思考遣词造句（还有把一些现成的文字搬运过来）。

1月

因考研而紧绷的弦暂时松了，1月3号，聚餐前还在赶制很水的课程报告，烧烤吃的很撑，回想起来，我自从转专业以后，经常做这种没什么意义，但还是要心理安慰式地打磨课程作业的事情。
1月8号，为了弥补以后可能的遗憾，独自逛了南京：南京美术馆，总统府。
然后很快回家，度过了最后一个漫长的寒假。

对10的想念，生根发芽。

2月

由于注销高中的银行卡，重游启东，和父母吃烤鱼，味道很好。
更加地想念10。

得知了惨淡的分数，并且很快地说服了自己，并不感到多么地遗憾。

3月

和过去的很多次一样，坐在教三的五楼，眺望窗外，思考人生，并且拍照记录美好的阳光。
同时由于南京疫情，开始封校生活。期间吃了很多顿美味的外卖。

愈发地思念10，翻遍其微博。

其实我也很想学摄影啊。

3月21日开会，当面见到了她。

4月

在疯狂的核酸中度过。
思考面试的事情，很痛苦。
经常地吃外卖，每次拿外卖都要走过21栋中间漫长的走廊，经常看见各种行人，不得不感叹：“没人永远年轻，但总有人年轻”。

5月

和舍友一起拍毕业照，摆了很傻的pose。

5月底，舍友桃率先离开。
在去学科楼开会的路上，记录下了后门口的灿烂阳光，

6月

只有失去了才会懂得珍惜。

在大学的最后时光，时间像指间流沙无法停留，任凭它流下。

被批评了毕业论文致辞太短，不曾用心，在思考毕业论文致辞的过程中，思考了很多。

囿于线上培训。

6月18号，在热辣的阳光下，最后站在母校的大门外，凝视了一阵。

在母校的时光，我会永远怀念。

回家，久违地感受到了家的感觉。在我看来，人脑的感觉也像是机器学习的过程，人脑从外界输入信息，得到情感的反馈，然后有时候得到类似的输入，又能得到类似的情感反馈。

补了颗牙。

站在老家天台拍了几张晚霞。

26日，站在启东中学的外面看了几眼。

6月28日，重回舅舅家，吃了一碗大排面。
同日，重游阔别了十余年的唐镇。

月底，开始了人生中第一次真正意义上的工作。

和一昊进行了更为深入的交流。

7月

7月底，奉贤的森林公园划船，还算是有意思的。

8月

8月27日，重游高桥。
重返大同路。高桥以古镇作为卖点，踏上了那座桥的台阶，拍了几张照片，虽然不是很专业。

8月

5号，第一次领到工资。

9月

工作不顺，怀疑自己工作的意义。

读书活动选了一批书，目的是拓展知识面，搞到更多的钱，以及找到自己人格中的不足，追根溯源，尝试做一个更好的人。

10月

期盼了三个月的国庆，到来，拔了一颗智齿。

心灵奇旅：人生中值得铭记的瞬间未必是功成名就的那一刻，而是平凡生活中的点点滴滴。

回上海时坐了人生中第一次火车。

得到50-25的消费券，吃了一顿很撑的麦当劳。

在高行的桥下转了一圈。

10月30日，拥有了一台很贵、很好的手机。
感觉欣喜、激动，但心理并未像想象中那样高兴。

11月

逐步轻车熟路。
出差了一次，成长了不少。

12月

瘟疫大逃杀。
